## Tools

You must have at least 3 tools. The three required tools are listed — add any additional tools below them.

### Tool 1: search_listings

**What it does:**
Filters the mock listing database by size and maximum price ceiling (if provided), then scores remaining items based on keyword matching against the user's natural language description to return a sorted list of relevant items.

**Input parameters:**

- `description` (str): Keywords describing what the user is looking for.
- `size` (str | None): Optional size string to filter by (case-insensitive substring match).
- `max_price` (float | None): Optional maximum price threshold (inclusive).

**What it returns:**
A list of matching listing dictionaries from `listings.json` sorted by relevance score (highest first).

**What happens if it fails or returns nothing:**
Returns an empty list `[]` without throwing an exception. The agent checks for this, sets an error in the session state, and halts execution early.

---

### Tool 2: suggest_outfit

**What it does:**
Leverages the Groq LLM to generate 1–2 complete outfit combinations pairing the newly selected item with pieces from the user's current wardrobe.

**Input parameters:**

- `new_item` (dict): The listing dictionary of the chosen secondhand piece.
- `wardrobe` (dict): The user's wardrobe dictionary containing a list of item dictionaries under the 'items' key.

**What it returns:**
A string containing structured outfit suggestions and specific styling advice.

**What happens if it fails or returns nothing:**
If the wardrobe is empty, it calls the LLM with a specialized prompt to provide general styling advice, aesthetic vibes, and recommended pairings for that item type instead of failing.

---

### Tool 3: create_fit_card

**What it does:**
Generates a short, high-temperature, social-media-ready caption (OOTD style) combining the new find and the outfit suggestion.

**Input parameters:**

- `outfit` (str): The text output string generated by `suggest_outfit`.
- `new_item` (dict): The listing dictionary of the chosen item.

**What it returns:**
A 2–4 sentence casual, punchy caption mentioning the item name, price, and platform naturally.

**What happens if it fails or returns nothing:**
If the input outfit or item data is missing or whitespace-only, it returns a descriptive error message string rather than raising an exception.

## Additional Tools (if any)

### Tool 4: compare_price

**What it does:**
Calculates the average price of items belonging to the same `category` within the mock listings dataset and evaluates whether the selected item's price represents a good value.

**Input parameters:**

- `new_item` (dict): The listing dictionary of the item being evaluated.

**What it returns:**
A dictionary containing statistical valuation metrics: `{"average_price": float, "deal_rating": str, "message": str}`.

**What happens if it fails or returns nothing:**
If no items match the category to create a baseline, it defaults gracefully to a "Fair Value" baseline assessment and allows the loop to pass code execution forward.

## Planning Loop

**How does your agent decide which tool to call next?**
The agent uses a deterministic waterfall execution loop with explicit validation gates:

1. **Extraction:** The user's query is passed to the Groq LLM using structured JSON output mode to extract parameters into `session["parsed"]`.
2. **Search:** `search_listings` is called. If `len(search_results) == 0`, the agent sets `session["error"]` and breaks out of the loop immediately.
3. **Appraisal:** If items are found, the top result is extracted and passed to `compare_price` to add pricing context to the state.
4. **Generation:** The item is passed to `suggest_outfit`, followed by `create_fit_card`, using information updated in the session state at each step.

---

## State Management

**How does information from one tool get passed to the next?**
State is tracked through a central session dictionary initialized at the start of `run_agent()`. Tools read from and write to this dictionary via clear keys:

````python

    "price_evaluation": dict | None,
    "wardrobe": dict,
    "outfit_suggestion": str | None,
    "fit_card": str | None,
    "error": str | None
}

## A Complete Interaction (Step by Step)

Example user query: "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

Step 1:
The raw user query is structured using Groq JSON mode into parsed arguments: {"description": "vintage graphic tee", "size": null, "max_price": 30.0}.

Step 2:
search_listings executes with the parsed arguments, scanning listings.json. It identifies lst_006 ("Graphic Tee — 2003 Tour Bootleg Style" at $24.00) as the top match and stores it in session["selected_item"].

Step 3:
compare_price calculates the average price of all "tops" in the dataset ($22.50) and tags the item as a fair deal since it's close to the baseline. This valuation is stored in session["price_evaluation"].

Step 4:
suggest_outfit takes the selected tee and pairs it explicitly with the "Baggy straight-leg jeans" and "Chunky white sneakers" found in the user's wardrobe file, updating session["outfit_suggestion"].

Step 5:
create_fit_card combines the item metadata and outfit text to return a ready-to-post, engaging social media caption string inside session["fit_card"].

Final output to user:
The user sees the top listing details ($24 shirt on Depop), the price verification message, a tailored outfit combination using their actual closet pieces, and an OOTD caption.
````
